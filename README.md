# seuchico

Este repositório contém o código-fonte do portfólio pessoal de Claudio Santos, um desenvolvedor em formação focado em desenvolvimento web, sistemas embarcados e aprendizado contínuo. O site serve como uma plataforma para documentar sua jornada técnica, projetos e evolução.

## Visão Geral do Projeto

O site `seuchico` é uma aplicação web estática construída com HTML, CSS e JavaScript puro. Ele apresenta as seguintes seções:

- **Home (`index.html`):** Introdução ao desenvolvedor e seus focos.
- **Sobre (`sobre.html`):** Detalhes sobre a jornada e motivações de Claudio.
- **Projetos (`projetos.html`):** Uma vitrine dos projetos desenvolvidos, como o SmartWearable e o Dashboard IoT.

## Tecnologias Utilizadas

- **HTML5:** Estrutura semântica das páginas web.
- **CSS3:** Estilização e responsividade, utilizando a fonte Poppins do Google Fonts.
- **JavaScript:** Pequenas interações, como o efeito de esconder o cabeçalho ao rolar a página.

## Como Rodar Localmente

Para visualizar o projeto localmente, siga os passos abaixo:

1.  **Clone o repositório:**
    ```bash
    git clone https://github.com/seuchco/seuchico.git
    ```
2.  **Navegue até o diretório do projeto:**
    ```bash
    cd seuchico
    ```
3.  **Abra os arquivos HTML:**
    Simplesmente abra `index.html`, `sobre.html` ou `projetos.html` em seu navegador web preferido.

## Contribuição e Melhorias Futuras

Este projeto está em constante evolução. As seguintes áreas foram identificadas para futuras melhorias e estão documentadas como issues no GitHub:

-   **Estrutura e Organização do Código:** Reduzir a duplicação de código (DRY) no cabeçalho e rodapé, possivelmente através de componentização com JavaScript ou uso de um gerador de sites estáticos.
-   **Performance e SEO:** Otimizar o carregamento de fontes, adicionar meta tags Open Graph e garantir que todos os scripts sejam carregados de forma não bloqueante e funcional em todas as páginas.
-   **Qualidade do CSS:** Corrigir erros de sintaxe e organizar melhor as regras CSS para maior manutenibilidade.
-   **Acessibilidade (A11y):** Melhorar a semântica e a navegação para usuários de leitores de tela, adicionando atributos `aria-label` e indicando o link ativo no menu.
-   **Experiência do Usuário (UX):** Refatorar a responsividade do cabeçalho para telas menores e aprimorar o feedback visual em elementos interativos.

Sinta-se à vontade para abrir novas issues ou enviar pull requests com sugestões e contribuições.
